# Rasgo User-Defined Transforms (UDTs)

Rasgo UDTs provide jinja SQL templates that can be applied to your data using rasgoQL, a pandas-like python package.
- UDTs are equivalent to SQL functions that accept a table or view from your eixsting DataWarehouse and return a SQL string to transform it
- Rasgo has built a starter library of transforms for you to use or fork
- *Coming Soon:* Users will be able to create their own UDTs and add them to a private namespace or contribute to the open-source library

## Running UDTs

Rasgo UDTs can be applied in your Rasgo Feature Store account ([pyrasgo](https://pypi.org/project/pyrasgo/) - Account required) or using the Rasgo open-source package ([rasgoql](https://pypi.org/project/rasgoql/) - totally free).

### Package Dependencies
- pyyaml

# Quick Start
```python
pip install -u rasgoql

# Connect RQL to your existing DataWarehouse
rql = rasgoql.connect()

transforms = rql.list_transforms()
print(transforms)

# Point RQL to a table in your Datawarehouse
ds = rql.dataset('<DATABASE.SCHEMA.TABLE>')

# Apply a Transform to the dataset and preview the results
ds2 = ds.train_test_split(
  order_by = ['DATE'],
  train_percent = .8
)
ds2.preview()

# Preview the SQL generated by this transform
ds2.sql

# Save the query to your DataWarehouse as a view
ds2.save(table_name='MY_RQL_TEST')
```
